# VMware忘记开机密码

1. 开机启动，按"e"进入编辑界面

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/b6dbcf4090e84599b451b8d3c8e483c0.png)

2. 进入编辑界面，使用键盘上的上下键把光标往下移动，找到以““Linux16”开头内容所在的行数”，在行的最后面输入（最好把前面的语言改为en_US不然后面改密码时提示语言是乱码）：init=/bin/sh

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/120071be3bea41cbb85b3954ef732f4b.png)

3. 输入完成后，按下CTRL+X进入单用户模式

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/760db567f22c41bdb8194e5d89739e49.png)

4. 输入以下代码

```
mount -o remount, rw /
LANG=en
passwd root
touch /.autorelabel
exec /sbin/init
```

进入下图界面进行代码输入的操作。需要两次输入相同的代码，之后输入最后一句代码进行重启。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/56ff39fd4fa14a56a922a8413d9ed021.png)

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/a6ccef51d70f4091a9e2e352eba0b351.png)

5. 最后重启完成登录root账号进行验证。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/d07b1606db2d41619abe89643c9e9c62.png)

# 文件挂载问题

```
docker run -d --name seata \   
-p 8091:8091   -p 7091:7091 \  
--privileged=true \
-v /usr/local/seata/conf/application.yml \
:/seata-server/resources/application.yml \
-e SEATA_IP=192.168.100.104   seataio/seata-server
```

这里的 -v 会把第一个参数的application.yml文件当做seata的配置文件启动，如果conf目录下的application.yml为空，则会使用一个空的application.yml文件启动seata，因此在启动seata之前，需要先

```
docker cp seata:/seata-server/resources/application.yml /usr/local/seata/conf/
```

把配置文件复制到conf目录下，然后再使用第一个命令启动seata
